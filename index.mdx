<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-3xl font-bold text-gray-900 dark:text-white mb-4">Quick start</h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">Get Retina running on your plugin in under a minute</p>
  
  <Steps>
    <Step title="Install Retina">
      On macOS and Linux, run the installation script:
      
      ```bash
      curl -sSL https://raw.githubusercontent.com/chinmay505/retina/main/install.sh | bash
      ```
      
      On Windows, clone the repository and install dependencies:
      
      ```bash
      git clone https://github.com/chinmay505/retina.git
      cd retina
      composer install --no-dev --optimize-autoloader
      ```
      
      <Note>Requires PHP 8.1 or higher and Composer</Note>
    </Step>
    
    <Step title="Scan your plugin">
      Navigate to your plugin directory and run Retina:
      
      ```bash
      retina run
      ```
      
      Or scan a specific directory:
      
      ```bash
      retina run /path/to/your/plugin
      ```
      
      Retina will analyze your plugin and generate a detailed report.
    </Step>
    
    <Step title="Review the results">
      By default, Retina creates a `retina-report.md` file with all detected issues:
      
      ```bash
      # View the report
      cat retina-report.md
      ```
      
      <Accordion title="Example output">
        ```markdown
        # Retina Analysis Report
        
        ## Summary
        - Total issues: 5
        - Error: 2
        - Warning: 3
        
        ## Issues
        
        ### src/Main.php:42
        **Category:** deprecated_api  
        **Severity:** warning
        
        Using deprecated method Player::getInventory()
        Use Player::getArmorInventory() instead
        ```
      </Accordion>
    </Step>
    
    <Step title="Configure for your workflow">
      Create a `retina.yml` config file to customize analysis:
      
      ```bash
      retina init
      ```
      
      This generates a configuration file where you can adjust strictness levels, exclude specific analyzers, and more. See the [configuration guide](/guides/configuration) for details.
    </Step>
  </Steps>
</div>
